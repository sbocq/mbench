mbench.benchmark

RuntimeConfig

class RuntimeConfig[-I, S] extends Config[I, Any, S]

A runtime configuration creates an new setup S for each input I of a benchmark before it evaluates the test over several runs. This means that the time it takes to create a setup (e.g. a thread pool) is not accounted in the execution times measured for a given test. For instance, for each input I of the benchmark:

  1. the benchmark creates a new setup by calling the setUp method, 2. the test is executed a number of times, as specified by the benchmark, and the its execution time is measured, 3. then, its runtime configuration is teared down using the tearDown method.

Tests that depend on a runtime configuration are created either using the runtime factory method, or the runtimeStatic methods of their companion object if they also depend on a static configuration (see Test, TestSeq or RuntimeConfig).

I

the input of the benchmark used to instantiate a new setup.

S

the type of setup instantiated by this configuration (e.g. an ExecutorService).
